# Программное обеспечение компьютерного моделирования квадрокоптера

Проект содержит первый вариант MATLAB-комплекса для оценки адекватности компьютерной модели квадрокоптера по данным бортового журнала ArduPilot.

Текущий уровень зрелости: **оценка адекватности компьютерной модели по данным бортовой системы регистрации**. Данная проверка выполнена по данным бортового журнала и не является полной независимой валидацией реального изделия по внешним средствам измерений.

## Назначение

Программное обеспечение компьютерного моделирования выполняет:

- чтение табличных данных бортовых журналов ArduPilot из CSV и MAT;
- подготовку данных для расчета компьютерной модели;
- построение модели угловых скоростей и модели движения с шестью степенями свободы;
- идентификацию параметров модели угловых скоростей по части данных;
- расчет компьютерной модели на отложенных участках;
- сопоставление расчетных и зарегистрированных параметров;
- формирование отчета оценки адекватности по данным бортового журнала.

## Почему это не полная независимая валидация

На первом этапе источником сведений является бортовая система регистрации изделия. Валидационный базис не включает независимые внешние средства измерений положения, скорости, ориентации, сил и моментов. Поэтому результат относится к проверке согласованности компьютерной модели с выбранными записями бортового журнала и не подтверждает полную валидацию реального изделия.

## Запуск MATLAB-проекта

В MATLAB выполните:

```matlab
run('matlab/startup.m')
run('matlab/run_all_tests.m')
```

Первый запуск автоматически создает конфигурацию изделия `config/izdeliye_qc_ardupilot_v0.json`.

## Подготовка CSV

Файл .BIN на первом этапе напрямую не разбирается. Бортовой журнал .BIN должен быть предварительно приведен к табличному виду штатными средствами пользователя. Поддерживаются:

- CSV;
- MAT;
- table MATLAB;
- timetable MATLAB.

Рекомендуемые столбцы описаны в [docs/dogovor_na_dannye.md](docs/dogovor_na_dannye.md). Отсутствие необязательных каналов отражается в отчете качества данных и не должно аварийно завершать выполнение.

## Идентификация параметров

Поместите CSV или MAT в `data/raw`, затем выполните:

```matlab
run('scripts/03_podgotovit_dannye.m')
run('scripts/04_identifikatsiya_modeli_uglovykh_skorostei.m')
```

Модель угловых скоростей использует состояние `omega = [p; q; r]`, вход `u = [u_roll; u_pitch; u_yaw]` и уравнение `domega/dt = A * omega + B * u + c`.

## Оценка адекватности

После подготовки данных и идентификации выполните:

```matlab
run('scripts/05_validatsiya_po_zhurnalu.m')
```

Отчет будет записан в `results/otchet_otsenki_adekvatnosti.md`. В отчете указываются участки идентификации и проверки, параметры модели, показатели расхождения, матрица валидации, предупреждения качества данных и границы применимости.

## Ограничения первого этапа

- Прямое чтение .BIN оставлено как заготовка адаптера.
- Произведения инерции CAD сохраняются, но по умолчанию не используются.
- Оси CAD не считаются совпадающими с осями тела ArduPilot без явного подтверждения.
- Противоречие винтов Gemfan 8060 и APC 7x11E сохраняется в конфигурации и отчетах.
- Критерии приемлемости являются предварительными и подлежат уточнению после накопления валидационного базиса.
- Расчетное ядро отделено от чтения данных, графиков и отчетов для дальнейшего переноса в Simulink.
