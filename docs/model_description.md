# Описание компьютерной модели

## Объект моделирования

Объект моделирования — квадрокоптер `qc_ardupilot_v0` со схемой QUAD/X, прошивкой ArduCopter V4.7.0-dev и полетным контроллером MatekSYS H743-SLIM V3.

## Состав компьютерной модели

Компьютерная модель первого этапа включает:

- модель угловых скоростей;
- модель движения с шестью степенями свободы;
- модель винтомоторной группы;
- модель аккумулятора начального уровня;
- слой преобразования координат CAD в связанную систему координат изделия.

## Модель угловых скоростей

Состояние: `omega = [p; q; r]`, рад/с.

Вход: `u = [u_roll; u_pitch; u_yaw]`.

Уравнение:

```text
domega/dt = A * omega + B * u + c
```

Внутренние единицы расчета — рад/с. Для отчетов допускается представление в град/с.

## Модель движения с шестью степенями свободы

Состояние включает положение, скорость, ориентацию, угловые скорости и тягу двигателей.

Вход модели — нормированные команды двигателей от 0 до 1.

Модель двигателя:

```text
T_cmd_i = kT * u_i^2
dT_i/dt = (T_cmd_i - T_i) / tau_motor
```

Силы включают тягу, силу тяжести и начальное линейное сопротивление воздуха.

Моменты:

```text
tau = sum(r_i x F_i) + spin_sign_i * kQ_over_kT * T_i
```

## Модель винтомоторной группы

Модель винтомоторной группы первого этапа задает коэффициент тяги, постоянную времени двигателя и отношение реактивного момента к тяге.

## Модель аккумулятора начального уровня

Модель аккумулятора на первом этапе является справочной частью конфигурации и хранит номинальное напряжение и емкость.

## Системы координат

CAD-координаты сохраняются в исходном виде. Матрица `CAD -> связанная система координат изделия` хранится явно. Оси CAD не считаются автоматически совпадающими со связанной системой координат изделия.

По умолчанию используется диагональная матрица инерции. Произведения инерции CAD сохраняются, но по умолчанию не применяются до подтверждения систем координат.

## Допущения

- Данные для оценки адекватности поступают из бортового журнала ArduPilot.
- Сопротивление воздуха задано моделью начального уровня.
- Критерии приемлемости являются предварительными.
- В исходном описании указаны винты Gemfan 8060, а в CAD-выгрузке присутствуют винты APC 7x11E.

## Ограничения применимости

Компьютерная модель применима для предварительной оценки адекватности по данным бортового журнала. Результат не является полной независимой валидацией реального изделия по внешним средствам измерений.
## Уточнение PR №3

`ModelRate` является компьютерной моделью первого уровня. Она применяется только для угловых скоростей `p`, `q`, `r` и используется для первой идентификации по данным бортового журнала.

`ModelRate` не является полной моделью движения изделия. Она не описывает поступательное движение, энергетику, аэродинамические силы и полную динамику винтомоторной группы.

`Model6DOF` является расчетной заготовкой для последующей настройки. Его идентификация по реальным бортовым журналам и сопоставление с ВБ будут выполняться после подтверждения участков, состава каналов и качества данных.

## Уточнение PR №4

`ModelRateContinuous` соответствует существующей непрерывной постановке:

```text
domega/dt = A * omega + B * u + c
```

`ModelRateDiscrete` добавлен как диагностический вариант:

```text
omega[k+1] = Ad * omega[k] + Bd * u[k-delay] + cd
```

Дискретная модель не заменяет основную компьютерную модель. Она нужна для диагностики временной синхронизации, задержки входа, обусловленности регрессии и применимости первого уровня модели угловых скоростей к данным бортового журнала.

Ограничения модели угловых скоростей первого уровня:

- модель описывает только `p`, `q`, `r`;
- модель не учитывает полную аэродинамику, поступательное движение и изменение режима работы винтомоторной группы;
- модель чувствительна к качеству участков identification и validation;
- модель не должна подгоняться по validation-участкам;
- отрицательный или неприменимый `FIT` является диагностическим результатом, а не основанием скрывать ограничения.

Данная проверка выполнена по данным бортового журнала и не является полной независимой валидацией реального изделия по внешним средствам измерений.

## Статусы параметров Model6DOF PR №8

В PR №8 для `Model6DOF` введены статусы:

- `motor_map_status`;
- `spin_sign_status`;
- `cad_to_body_status`;
- `propulsion_model_status`;
- `propeller_configuration_status`.

Статусы разделяют физически подтвержденные данные и расчетные гипотезы. Если физическая проверка отсутствует, motor map, spin signs и `CAD -> body` остаются в состоянии `hypothesis_from_log`.

Первичная модель тяги ВМГ описывается квадратичной зависимостью:

```text
T = a0 + a1*u + a2*u^2
```

Модель тока имеет вид:

```text
I = b0 + b1*u + b2*u^2
```

Модель внутреннего сопротивления батареи оценивается по зависимости:

```text
V = V0 - I*R_internal
```

Без ESC telemetry и стендовых данных параметры тяги, тока и батареи не считаются подтвержденными. Противоречие `Gemfan 8060` и `APC 7x11E` сохраняется до физической проверки установленного винта.

До физического подтверждения подключения моторов, направлений вращения и установленного типа винтов выбранная конфигурация Model6DOF должна рассматриваться как расчетная гипотеза.

## Демонстрационный режим Model6DOF PR №5

`Model6DOF demonstration mode` предназначен для проверки работоспособности решателя, примитивного регулятора и постпроцессора. В этом режиме модель выполняет висение, набор высоты и полет по прямоугольнику.

Примитивный регулятор включает:

- внешний контур положения `PrimitivePositionController`;
- контур высоты `PrimitiveAltitudeController`;
- контур углов `PrimitiveAttitudeController`;
- смеситель `quadXMixer` для схемы QUAD/X.

Траектория `box trajectory` содержит взлет, висение, четыре стороны прямоугольника, повторное висение и посадку. Она используется как демонстрационный сценарий для проверки, что компьютерная модель способна выполнить устойчивый расчет движения.

Ограничения демонстрационного режима:

- геометрия для устойчивого demo-полета использует симметричную QUAD/X-раму, потому что соответствие CAD-осей и связанной системы координат еще не подтверждено;
- параметры тяги заданы демонстрационно и подлежат идентификации;
- полноценная аэродинамика не реализована;
- ESC telemetry в реальных журналах отсутствует;
- результат не подтверждает адекватность модели реальному изделию.

Демонстрационный полет Model6DOF в PR №5 предназначен для проверки работоспособности решателя, регулятора и постпроцессора. Он не подтверждает адекватность модели реальному изделию.

## Log replay mode для Model6DOF PR №6

В PR №6 добавлен режим replay-расчета `Model6DOF` по коротким окнам бортового журнала. Препроцессор формирует начальное состояние по `ATT`, `RATE`, высоте и скорости, а входы двигателей строятся по `RCOU.C1`...`RCOU.C4`.

В первичном подборе оцениваются:

- `thrust_scale`;
- `motor_tau_s`;
- `kQ_over_kT`;
- `linear_drag_x`;
- `linear_drag_y`;
- `linear_drag_z`.

Фиксированными остаются масса изделия, диагональная матрица инерции и запрет использования произведений инерции до подтверждения соответствия систем координат. Режим replay является инструментом настройки и диагностики компьютерной модели, но не является полной независимой валидацией реального изделия.

## Калибровка мотор-маппинга и систем координат PR №7

Для `Model6DOF` добавлен диагностический слой базовой физической конфигурации. Он проверяет, как каналы `RCOU.C1`...`RCOU.C4` соответствуют физическим моторам, какие знаки вращения используются для момента рыскания и как CAD-координаты переводятся в связанную систему координат изделия.

Матрица распределения QUAD/X связывает тяги моторов с вектором:

```text
[суммарная тяга; момент крена; момент тангажа; момент рыскания]
```

Знаки моментов документированы в функции `buildQuadXAllocationMatrix`. Преобразование `CAD -> body` проверяется набором кандидатных матриц, но по умолчанию произведения инерции CAD не используются до подтверждения систем координат.

Поля конфигурации разделяются на:

- подтвержденные исходными данными изделия;
- извлеченные из параметров ArduPilot `PARM`;
- расчетные гипотезы, полученные по бортовому журналу;
- неподтвержденные физически.

Выбранный мотор-маппинг и преобразование `CAD -> body` являются расчетной гипотезой, сформированной по бортовым журналам. Для окончательного подтверждения требуется физическая проверка подключения моторов, направления вращения и системы координат изделия.

Данная проверка выполнена по данным бортового журнала и не является полной независимой валидацией реального изделия по внешним средствам измерений.
