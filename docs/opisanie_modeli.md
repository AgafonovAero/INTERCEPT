# Описание компьютерной модели

## Объект моделирования

Объект моделирования: квадрокоптер `qc_ardupilot_v0` со схемой QUAD/X, прошивкой ArduCopter V4.7.0-dev и полетным контроллером MatekSYS H743-SLIM V3.

Известные исходные данные включают массу по CAD 3.269 кг, центр масс CAD, моменты инерции CAD, координаты двигателей CAD, сведения о двигателях, регуляторе двигателей и аккумуляторной батарее.

В исходном описании указаны винты Gemfan 8060, а в CAD-выгрузке присутствуют винты APC 7x11E. Это противоречие не устраняется автоматически и должно учитываться при оценке качества исходных данных.

## Состав модели

Компьютерная модель первого этапа включает:

- модель угловых скоростей;
- модель движения с шестью степенями свободы;
- модель винтомоторной группы первого уровня;
- справочную модель аккумуляторной батареи;
- явный слой преобразования координат CAD в связанную систему координат изделия.

## Модель угловых скоростей

Состояние:

```text
omega = [p; q; r], рад/с
```

Вход:

```text
u = [u_roll; u_pitch; u_yaw]
```

Уравнение:

```text
domega/dt = A * omega + B * u + c
```

Матрица `A` описывает демпфирование, матрица `B` описывает влияние управляющих воздействий, вектор `c` описывает постоянное смещение. В первом варианте параметры оцениваются отдельно для крена, тангажа и рыскания.

## Модель движения с шестью степенями свободы

Состояние включает:

- положение;
- скорость;
- ориентацию;
- угловые скорости;
- состояния тяги двигателей.

Вход модели: нормированные команды двигателей `u_i` от 0 до 1.

Модель тяги двигателя:

```text
T_cmd_i = kT * u_i^2
dT_i/dt = (T_cmd_i - T_i) / tau_motor
```

Силы:

- тяга винтомоторных групп;
- сила тяжести;
- линейное сопротивление воздуха начального уровня.

Моменты:

```text
tau = сумма(r_i x F_i) + знак_вращения_i * kQ_over_kT * T_i
```

## Системы координат

CAD-координаты сохраняются в исходном виде. В конфигурации хранится матрица преобразования `CAD -> связанная система координат изделия`. По умолчанию используется единичная матрица, но это не является подтверждением совпадения осей.

По умолчанию используется диагональная матрица инерции:

```text
diag([Ixx_cad_raw, Iyy_cad_raw, Izz_cad_raw])
```

Произведения инерции сохраняются в конфигурации, но не используются до подтверждения соответствия систем координат. Параметр `use_products_of_inertia` по умолчанию равен `false`.

## Допущения

- Валидационный базис первого уровня составляют выбранные бортовые журналы ArduPilot.
- Сопротивление воздуха задано линейной моделью начального уровня.
- Тяга двигателя пропорциональна квадрату нормированной команды.
- Задержка входа в модели угловых скоростей подбирается целым числом шагов дискретизации.

## Границы применимости

Компьютерная модель применима для предварительного расчета и оценки адекватности по данным бортового журнала. Она не предназначена для заявления полной независимой валидации реального изделия без внешних средств измерений.
